# Tailwind Next.js Starter Blog을 GitHub Pages에 배포하기

이 글에서는 **Tailwind Next.js Starter Blog** 템플릿을 사용하여 개인 블로그를 만들고, GitHub Pages를 통해 배포하는 방법을 단계별로 설명합니다.

---

## 1. 프로젝트 준비

### 1.1 Tailwind Next.js Starter Blog 복제
먼저, 템플릿 소스를 로컬로 복제합니다:

```bash
git clone https://github.com/timlrx/tailwind-nextjs-starter-blog.git my-blog
cd my-blog
```

### 1.2 의존성 설치
프로젝트 디렉토리로 이동한 후 필요한 의존성을 설치합니다:

```bash
npm install
```
또는
```bash
yarn install
```

### 1.3 로컬에서 프로젝트 실행

```bash
npm run dev
```

이후 브라우저에서 [http://localhost:3000](http://localhost:3000)을 열어 블로그를 확인합니다.

---

## 2. GitHub Repository 설정

### 2.1 GitHub Repository 생성
GitHub에서 새로운 저장소를 생성합니다. 저장소 이름은 원하는 대로 설정하세요(예: `my-blog`).

### 2.2 로컬 프로젝트와 GitHub 연결
GitHub 저장소를 로컬 프로젝트에 연결합니다:

```bash
git remote add origin https://github.com/<your-username>/my-blog.git
git branch -M main
git push -u origin main
```

---

## 3. GitHub Pages 설정

### 3.1 `next.config.js` 수정
GitHub Pages는 정적 파일만 지원하므로, Next.js 프로젝트를 정적 사이트로 변환해야 합니다. 프로젝트의 `next.config.js` 파일을 다음과 같이 수정합니다:

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
  assetPrefix: process.env.NODE_ENV === 'production' ? '/my-blog/' : '',
  basePath: process.env.NODE_ENV === 'production' ? '/my-blog' : '',
};

module.exports = nextConfig;
```

### 3.2 `package.json` 수정
배포 명령어를 추가합니다:

```json
"scripts": {
  "build": "next build && next export"
}
```

### 3.3 GitHub Actions 설정
`.github/workflows/deploy.yml` 파일을 생성하고 다음 내용을 추가합니다:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Install dependencies
        run: npm install

      - name: Build project
        run: npm run build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./out
```

### 3.4 GitHub Pages 활성화
1. GitHub Repository의 **Settings > Pages**로 이동합니다.
2. **Source**를 `gh-pages` 브랜치로 설정합니다.
3. 변경 사항을 저장합니다.

---

## 4. 배포 테스트

1. 로컬에서 변경 사항을 커밋하고 푸시합니다:

   ```bash
   git add .
   git commit -m "Set up GitHub Pages deployment"
   git push origin main
   ```

2. GitHub Actions 탭에서 워크플로우가 성공적으로 실행되는지 확인합니다.
3. GitHub Pages URL(예: `https://<your-username>.github.io/my-blog/`)에서 블로그가 정상적으로 표시되는지 확인합니다.
