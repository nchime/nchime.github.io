### 소개

[Tailwind Next.js Starter Blog](https://github.com/timlrx/tailwind-nextjs-starter-blog)는 Tailwind CSS, MDX, Next.js를 기반으로 간단하고 스타일리시한 블로그를 구축할 수 있는 오픈소스 프로젝트입니다. 이 글에서는 해당 템플릿을 사용해 블로그를 설정하고 GitHub와 Vercel을 활용해 배포하는 방법을 단계별로 소개합니다.

---

### 프로젝트 클론 및 초기 설정

1. **레포지토리 클론**
   GitHub에서 Starter Blog 프로젝트를 클론합니다.

   ```bash
   git clone https://github.com/timlrx/tailwind-nextjs-starter-blog.git my-blog
   cd my-blog
   ```

2. **의존성 설치**
   npm 또는 yarn을 사용해 의존성을 설치합니다.

   ```bash
   # npm 사용
   npm install

   # 또는 yarn 사용
   yarn install
   ```

3. **개발 서버 실행**
   프로젝트가 정상적으로 작동하는지 확인합니다.

   ```bash
   npm run dev
   ```

   `http://localhost:3000`에서 블로그를 확인할 수 있습니다.

4. **환경 파일 설정**
   필요한 경우 `.env.local` 파일을 생성해 환경 변수를 설정합니다. 기본적으로는 설정 없이도 작동합니다.

---

### 블로그 커스터마이징

1. **사이트 정보 수정**
   `data/siteMetadata.js` 파일에서 사이트 이름, 설명, 소셜 링크 등을 변경합니다.

   ```javascript
   module.exports = {
     title: 'My Blog',
     author: 'Your Name',
     description: 'A blog built with Next.js and Tailwind CSS',
     language: 'en-us',
     siteUrl: 'https://your-site-url.com',
     socialBanner: '/static/images/twitter-card.png',
   };
   ```

2. **블로그 포스트 작성**
   `data/blog` 폴더에 새로운 MDX 파일을 추가해 포스트를 작성합니다.

   ```mdx
   ---
   title: 'Welcome to My Blog'
   date: '2025-01-01'
   tags: ['nextjs', 'tailwind']
   draft: false
   summary: 'This is my first post!'
   ---
   내용은 여기에 작성합니다.
   ```

3. **스타일 변경**
   Tailwind CSS를 사용해 `styles` 폴더에서 스타일을 수정하거나 Tailwind 클래스를 사용해 컴포넌트를 커스터마이징합니다.

### GitHub에 저장소 생성 및 코드 푸시

1. **GitHub 저장소 생성**
   GitHub에서 새 저장소를 만듭니다.

2. **코드 푸시**
   프로젝트를 GitHub 저장소에 푸시합니다.

   ```bash
   git init
   git remote add origin https://github.com/your-username/your-repo.git
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git push -u origin main
   ```

---

### Vercel로 배포하기

1. **Vercel 계정 생성 및 로그인**
   [Vercel](https://vercel.com/)에 가입하거나 로그인합니다.

2. **새 프로젝트 추가**
   1. Vercel 대시보드에서 **Add New Project**를 클릭합니다.
   2. GitHub 계정을 연결한 후, 방금 만든 저장소를 선택합니다.

3. **설정 확인**
   배포 설정에서 빌드 명령과 출력 디렉토리를 확인합니다. 기본 설정은 다음과 같습니다.
   - 빌드 명령: `npm run build`
   - 출력 디렉토리: `out`

4. **배포**
   설정을 완료하면 자동으로 배포가 시작됩니다. 완료되면 Vercel에서 제공한 URL에서 배포된 블로그를 확인할 수 있습니다.
